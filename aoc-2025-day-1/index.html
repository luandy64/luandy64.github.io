    AOC 2025 Day 1

    Page created : 2025-12-02T22:14:15-05:00
    Last updated : 2025-12-02T23:57:33-05:00

    Here’s a quick writeup of how I approached the problem.

    Part 1

    The first part described how we want to count the number of times the dial lands on zero after some set of turn instructions. So as long as we track the state of applying the turns to some inital state, we can count the number of times zero appears. I’m assuming people also take the example input to sketch a solution against. So I start by defing the example in one long string. This also clears up the initial input parsing logic. So here’s the input:

    (def test-1 "L68
  L30
  R48
  L5
  R60
  L55
  L1
  L99
  R14
  L82")

    As far as the input parsing goes, I wanted to take advantage of destructuring to split the pieces of each line.

    (->> test-1
     str/split-lines
     (mapv str/trim))
;; => ["L68" "L30" "R48" "L5" "R60" "L55" "L1" "L99" "R14" "L82"]

    Taking just the first few elements of the vector, here’s what the destructuring gives me.

    (->> test-1
     str/split-lines
     (mapv str/trim)
     (take 3)
     (mapv (fn [[dir & num]]
             {:dir dir
              :num num})))
;; => [{:dir \L, :num (\6 \8)}
;;     {:dir \L, :num (\3 \0)}
;;     {:dir \R, :num (\4 \8)}]

    My intention is to change the num from a list of characters into an integer and then to make dir decide if that integer is positive or negative. Using the same few inputs from above, this looks like:

    (->> test-1
     str/split-lines
     (mapv str/trim)
     (take 3)
     (mapv (fn [[dir & num]]
             (let [int-num (->> num
                                (apply str)
                                Integer/parseInt)]
               (if (= "L"
                      (str dir))
                 (* -1 int-num)
                 int-num)))))
;; => [-68 -30 48]

    This can be a little denser written with a cond->>

    (->> test-1
     str/split-lines
     (mapv str/trim)
     (take 3)
     (mapv (fn [[dir & num]]
             (cond->> (->> num
                           (apply str)
                           Integer/parseInt)
               (= "L" (str dir)) (* -1)))))
;; => [-68 -30 48]

    But it could be argued that the cond->> and ->> are a little redundant. But this comes at the price of using something truthy to force the (apply str) to run and to have the Integer/parseInt run.

      (->> test-1
       str/split-lines
       (mapv str/trim)
       (take 3)
       (mapv (fn [[dir & num]]
               (cond->> num
                 :make-str         (apply str)
                 :make-int         Integer/parseInt
                 (= "L" (str dir)) (* -1)))))
  ;; => [-68 -30 48]

    Yeah, I did all of this just to avoid the

    (if thingy
    (change blah)
    blah)

    Now that I have all of the inputs translated to negative and positive integers, I can just do clock-math on some starting position. The problem statement says the dial starts at 50, so I’ll start there.

    (->> test-1
     str/split-lines
     (mapv str/trim)
     (mapv (fn [[dir & num]]
             (cond->> num
               :make-str         (apply str)
               :make-int         Integer/parseInt
               (= "L" (str dir)) (* -1))))
     (reduce (fn [curr turn]
               (-> turn
                   (+ curr)
                   (mod 100)))
             50))
;; => 32

    This magically matches the value that the walkthrough of the example ends on. So the logic can’t be too far off. But this doesn’t give me the answer I want. Thankfully, there is a function in the standard library to return a sequence of the values that the accumulator takes on while processing some sequence: reductions.

    (->> test-1
     str/split-lines
     (mapv str/trim)
     (mapv (fn [[dir & num]]
             (cond->> num
               :make-str         (apply str)
               :make-int         Integer/parseInt
               (= "L" (str dir)) (* -1))))
     (reductions (fn [curr turn]
                   (-> turn
                       (+ curr)
                       (mod 100)))
                 50))
;; => (50 82 52 0 95 55 0 99 0 14 32)

    Just by switching the function out (and cleaning out the spacing a little bit) you can see we are able to get the answer to the example input. This obviously won’t work to manually count up all the zeros when the input is who-knows-how-long. So my final answer to part 1 is to tack on a filter and a count.

    (->> test-1
     str/split-lines
     (mapv str/trim)
     (mapv (fn [[dir & num]]
             (cond->> num
               :make-str         (apply str)
               :make-int         Integer/parseInt
               (= "L" (str dir)) (* -1))))
     (reductions (fn [curr turn]
                   (-> turn
                       (+ curr)
                       (mod 100)))
                 50)
     (filter zero?)
     count)
;; => 3

    Part 2

    My original approach to this part was to try and come up with rules to decide if I can deduce whether zero had been crossed based on where I started, where I ended, the direction of the turn, and the amount I turned. I still think this is a viable approach, but I could not think through it tonight.

    So the approach I took was to literally simulate the turning and literally track whether the current position is at zero or not.

    Probably the most interesting part to me is that I forget about into as a means of combining sequences. Especially vectors when I want to preserve the vector and not end up with a list. Other than that, I find it to be hilarious that I turned all of the input into some long list of -1 and 1 and processed every last one.

    (->> test-1
     str/split-lines
     (mapv str/trim)
     (reduce (fn [moves [dir & num]]
               (let [int-num (->> num
                                  (apply str)
                                  Integer/parseInt)
                     new-moves (repeatedly int-num
                                           #(if (= "L"
                                                   (str dir))
                                              -1
                                              1))]
                 (into moves new-moves)))
             [])
     (reduce (fn [{:keys [curr] :as state} move]
               (let [new-curr (mod (+ curr move)
                                   100)]
                 (-> state
                     (assoc :curr new-curr)
                     (update :hits #(if (zero? new-curr) (inc %) %)))))
             {:curr 50
              :hits 0}))

  (let [moves (->> day-1
                   #_test-1
                   str/split-lines
                   (mapv str/trim)
                   (mapv (fn [[dir & num]]
                           (cond->> num
                             :make-str     (apply str)

    (->> (mapv vector
               moves
               (partition 2 1 poss))
         (mapv (fn [[move [start end]]]
                 (let [abs-move (+ start move)
                       context [:start start
                                :move move
                                :end end
                                :abs abs-move]]
                   (or (and (zero? end)
                            1
                            #_:zero)
                       (and (< start (abs move))
                            (not (zero? start))
                            #_[:move-left-over-zero context
                               (let [laps (quot abs-move 100)]
                                 (if (zero? laps)
                                   1
                                   laps))]
                            (let [laps (quot abs-move 100)]
                              (if (zero? laps)
                                1
                                laps))
                            #_(or (quot abs-move 100)
                                  1))
                       (and (< end start)
                            (pos? move)
                            #_[:move-right-over-zero context
                               (let [laps (quot abs-move 100)]
                                 (if (zero? laps)

                                   laps))

                            (let [laps (quot abs-move 100)]

                                1
                                laps))
                            #_(or (quot abs-move 100)

                            (not (neg? start))
                            (not (zero? start))
                            #_[:neg-abs-move context]
                            (let [laps (quot abs-move 100)]

                                1
                                laps))
                            #_(or (quot abs-move 100)
                                  1))
